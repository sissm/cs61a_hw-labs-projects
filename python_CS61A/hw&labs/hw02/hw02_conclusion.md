# hw02_conclusion

## 学习总结

这次 `hw02` 主要练习了高阶函数、抽象、以及“函数返回函数”的写法。我对 Python 的理解更进一步，尤其是下面几个点：

- `product` 让我理解了“连续相乘”这种递归/迭代模式。
- `accumulate` 把“求和”“求积”等不同问题统一成了一个通用模板。
- `make_repeater` 让我真正接触到“返回一个函数”的写法。
- 我开始理解：函数不仅可以接收函数，也可以返回函数。

## 这次作业的核心收获

### 1. `product`

`product` 的作用是把前 `n` 项连续相乘。它让我明白，很多题目其实都可以写成“初始化一个结果，然后循环更新”的形式。

### 2. `accumulate`

`accumulate` 是一个更抽象的版本，它把“怎么融合”和“每一项是什么”分开了。这样一来，求和、求积、甚至更复杂的组合方式都可以复用同一个结构。

### 3. `make_repeater`

`make_repeater` 是我这次最重要的收获之一。它返回的是一个新函数，这个新函数会把原来的函数重复执行多次。这个过程让我更清楚地理解了：

- 函数可以作为参数传入
- 函数也可以作为结果返回
- 内层函数会记住外层变量，这就是闭包的雏形

## 最终结论

`hw02` 让我从“写普通函数”迈向了“把函数当作工具来组合”的阶段。相比之前只会写具体逻辑，现在我更能看懂 Python 里“抽象”和“复用”的价值。以后遇到类似题目，我会先想：这个问题能不能拆成更通用的模式，再用高阶函数把它表达出来。
